# PhotonLink Setup Guide

## Prerequisites

| Tool | Version | Required |
|------|---------|----------|
| Flutter SDK | ≥ 3.22 (Dart ≥ 3.3) | Yes |
| Android Studio | Latest | Yes (Android first) |
| Android SDK | API 34, min API 21 | Yes |
| Rust toolchain | ≥ 1.70 | Optional (Phase 8 Rust core) |
| flutter_rust_bridge_codegen | ≥ 2.11 | Optional (Phase 8 FFI) |
| Xcode | Latest | Optional (iOS future) |

## First-Time Setup

### 1. Install Flutter

Follow the official guide: https://docs.flutter.dev/get-started/install

Verify installation:

```bash
flutter doctor
```

### 2. Initialize Platform Files (if needed)

If platform folders are incomplete, run from the `photonlink_app/` directory:

```bash
cd photonlink_app
flutter create . --project-name photonlink_app --org com.photonlink
```

This merges missing platform scaffolding (icons, Gradle wrapper, etc.) without overwriting `lib/`.

### 3. Install Dependencies

```bash
cd photonlink_app
flutter pub get
```

### 4. Run on Android

Connect a device or start an emulator, then:

```bash
flutter run
```

### 5. Run Tests & Analysis

```bash
flutter analyze
flutter test
```

## Rust Core (Phase 8)

The Rust crate at `photonlink_core/` contains performance-critical transfer logic. It is **not required** for the app to run — the Dart backend is active by default.

### Build Rust Core

```bash
# Install Rust (if not installed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

cd photonlink_core
cargo test
cargo build --release
```

### Generate Flutter Rust Bridge Bindings

```bash
cargo install flutter_rust_bridge_codegen

cd photonlink_app
flutter_rust_bridge_codegen generate
flutter pub get
```

### Activate Rust Backend

After FRB codegen and successful `cargo build`:

1. Create `FrbCoreApi` adapter implementing `PhotonLinkCoreApi`
2. Override `photonLinkCoreApiProvider` in `bootstrap.dart`
3. Set `backendProvider` to `CoreBackend.rust` in `core_providers.dart`

See [FFI_DOCUMENTATION.md](FFI_DOCUMENTATION.md) for details.

## Troubleshooting

### Camera permission denied
Grant camera permission in device Settings, or tap "Open Settings" on the scan screen.

### Gradle build fails
Ensure `local.properties` contains your Android SDK path (auto-generated by Flutter).

### Missing platform icons
Run `flutter create .` as described in step 2.

### Rust backend not connected
Expected if Rust toolchain is not installed. App uses Dart backend automatically. Error: `RustCoreNotConnectedException` means FRB codegen has not been run.

## Current Limitations

- Rust backend requires toolchain + FRB codegen (Dart backend active by default)
- LZ4 compression disabled in Dart (available in Rust when activated)
- Optical Stream and Audio transports are UI stubs only
- Color Matrix uses one-way cyclic broadcast (no ACK/NAK feedback channel)

See [MIGRATION_REPORT.md](MIGRATION_REPORT.md) and [PHASE9_READINESS.md](PHASE9_READINESS.md).
